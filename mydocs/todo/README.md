## Contribution Guidelines

### Adding New Features

1. Create a new markdown file, naming format: `number-feature-name.md`
2. Include the following sections in the file:
   - Feature description
   - Core features
   - Data structure
   - Command interface
   - Medical safety principles

3. Update this README.md, adding the new feature to the table of contents

### Updating Existing Features

1. Directly edit the corresponding markdown file
2. Update the feature status and priority
3. Add an update log at the end of the file

### Status Labels

- 📝 Pending - Feature proposal stage
- 🔨 In Development - Currently being implemented
- ✅ Completed - Feature has been implemented
- ⏸️ On Hold - Temporarily shelved
- ❌ Cancelled - No longer planned for implementation

---

## Documentation Standards

Each feature proposal document should include the following sections:

1. **Feature Description** - Brief explanation of the feature's purpose and value
2. **Core Features** - List of main feature points
3. **Data Structure** - Data model in JSON format
4. **Command Interface** - CLI command examples
5. **Usage Examples** - Typical use scenarios
6. **Medical Safety Principles** - Relevant safety restrictions
7. **Notes** - Special instructions
8. **Reference Resources** - Links to related documentation
